# Medium Publishing Checklist

Use this checklist before publishing your article to ensure everything is ready.

---

## ‚úÖ Pre-Publishing Checklist

### Content Review

- [ ] **Replaced all placeholders**
  - [ ] `[Your Name]` ‚Üí Your actual name
  - [ ] `[Your Company]` ‚Üí Your company or "my organization"
  - [ ] `[@yourusername]` ‚Üí Your Medium username
  - [ ] `[@yourhandle]` ‚Üí Your Twitter handle
  - [ ] `[Your Profile]` ‚Üí Your LinkedIn/GitHub URL
  - [ ] `[hobby/interest]` ‚Üí Something you enjoy
  - [ ] `[Your Title]` ‚Üí Your job title
  - [ ] `[X] years` ‚Üí Your years of experience
  - [ ] `[he/she/they]` ‚Üí Your preferred pronoun

- [ ] **Read the full article at least once**
  - [ ] No obvious typos
  - [ ] Sentences flow naturally
  - [ ] Code examples are correct
  - [ ] Links work (test each one)

- [ ] **Technical accuracy**
  - [ ] Commands are correct
  - [ ] File paths match your project
  - [ ] Installation instructions work
  - [ ] GitHub links point to right repo

- [ ] **Story personalization**
  - [ ] Opening story feels authentic
  - [ ] Examples match your experience (or are generic enough)
  - [ ] Metrics are realistic (or clearly examples)
  - [ ] Tone matches your voice

### Formatting & Structure

- [ ] **Title is engaging**
  - [ ] Clear and specific
  - [ ] Includes key benefit
  - [ ] Under 60 characters (ideal)
  - [ ] No clickbait, but interesting

- [ ] **Subtitle is descriptive**
  - [ ] Complements the title
  - [ ] Sets expectations
  - [ ] Under 140 characters

- [ ] **Headings are clear**
  - [ ] Properly formatted (##, ###)
  - [ ] Logical hierarchy
  - [ ] Easy to scan

- [ ] **Code blocks formatted**
  - [ ] Language specified (\`\`\`javascript, \`\`\`bash, etc.)
  - [ ] Syntax highlighting works
  - [ ] No truncated lines
  - [ ] Comments explain complex parts

- [ ] **Lists are consistent**
  - [ ] Bullets or numbers used appropriately
  - [ ] Parallel structure
  - [ ] Not too long (break up if needed)

- [ ] **Paragraphs are readable**
  - [ ] 3-5 lines max
  - [ ] One idea per paragraph
  - [ ] White space between sections

### Visual Content

- [ ] **Cover image ready**
  - [ ] Dimensions: 1500 x 840 pixels (recommended)
  - [ ] High quality (not blurry)
  - [ ] Relevant to article
  - [ ] Text is readable
  - [ ] **Minimum requirement for publishing**

- [ ] **Inline images (optional but recommended)**
  - [ ] Architecture diagram
  - [ ] Terminal screenshots
  - [ ] Web UI screenshot
  - [ ] Charts/graphs if you have data

- [ ] **Image captions**
  - [ ] Each image has descriptive caption
  - [ ] Captions explain what's shown
  - [ ] Credits if using others' images

- [ ] **Image placement**
  - [ ] Images break up long text sections
  - [ ] Placed near relevant content
  - [ ] Not too many back-to-back

### SEO & Discovery

- [ ] **Tags selected (5 maximum)**
  - [ ] Primary: `API Design`
  - [ ] Secondary: `Developer Tools`
  - [ ] Tech stack: `Node.js`
  - [ ] Domain: `Microservices`
  - [ ] Trending: `Artificial Intelligence`

- [ ] **Custom excerpt written** (optional)
  - [ ] 140-160 characters
  - [ ] Summarizes key value
  - [ ] Used for social media previews

- [ ] **Links included**
  - [ ] GitHub repository
  - [ ] Your profile/portfolio
  - [ ] Related resources
  - [ ] All links use HTTPS

### Settings Configuration

- [ ] **Story settings reviewed**
  - [ ] Allow responses: **ON** (to get feedback)
  - [ ] Featured image: Set to cover image
  - [ ] Custom excerpt: Added (or using default)
  - [ ] Canonical URL: Empty (unless republishing)

- [ ] **Distribution chosen**
  - [ ] Option 1: "Distribute to Medium" (recommended for first article)
  - [ ] Option 2: "Followers only" (if you have established audience)

- [ ] **Social sharing configured**
  - [ ] Twitter auto-post (if you want)
  - [ ] LinkedIn auto-post (if you want)
  - [ ] Or manual sharing planned

## üì± Preview & Test

### Desktop Preview

- [ ] **Click "Preview" in Medium editor**
- [ ] **Check top to bottom:**
  - [ ] Title looks good
  - [ ] Cover image displays correctly
  - [ ] Headings are properly sized
  - [ ] Code blocks are readable
  - [ ] Lists are formatted
  - [ ] Images load
  - [ ] Links are clickable
  - [ ] No weird formatting

### Mobile Preview

- [ ] **Resize browser window (narrow width)**
- [ ] **Or use browser dev tools (F12 ‚Üí Mobile view)**
- [ ] **Check:**
  - [ ] Text is readable
  - [ ] Code blocks don't overflow
  - [ ] Images scale properly
  - [ ] No horizontal scrolling

### Grammar & Spell Check

- [ ] **Use Medium's built-in spell check**
- [ ] **Read aloud (catches awkward phrasing)**
- [ ] **Check common mistakes:**
  - [ ] Its vs it's
  - [ ] Your vs you're
  - [ ] Their vs there vs they're
  - [ ] Then vs than

---

## üöÄ Publishing

### Final Review

- [ ] **Re-read the article one more time**
- [ ] **Check all placeholders replaced**
- [ ] **Verify cover image uploaded**
- [ ] **Confirm tags added**
- [ ] **Review story settings**

### Hit Publish

- [ ] **Click "Publish" button**
- [ ] **Confirm distribution choice**
- [ ] **Double-check publication settings**
- [ ] **Click "Publish now"**

### Immediately After Publishing

- [ ] **Copy article URL**
- [ ] **Save URL somewhere safe**
- [ ] **Share on social media** (within first hour for best engagement)
  - [ ] Twitter/X
  - [ ] LinkedIn
  - [ ] Dev communities (Reddit, dev.to, Hacker News)
  - [ ] Team Slack/Discord

---

## üìä Post-Publishing

### First 24 Hours

- [ ] **Monitor comments**
  - [ ] Respond to every comment
  - [ ] Thank people for feedback
  - [ ] Answer questions
  - [ ] Fix typos if reported

- [ ] **Check stats**
  - [ ] Click avatar ‚Üí "Stats"
  - [ ] Note initial views
  - [ ] Check read ratio (40%+ is good)
  - [ ] See which sections people read

- [ ] **Share again**
  - [ ] Different angle on Twitter
  - [ ] LinkedIn post with professional context
  - [ ] Email to friends/colleagues

### First Week

- [ ] **Update GitHub README**
  - [ ] Add link to Medium article
  - [ ] Add "Featured Article" section
  - [ ] Update shields/badges

- [ ] **Engage with readers**
  - [ ] Respond to all comments
  - [ ] Share interesting discussions
  - [ ] Answer DMs about the article

- [ ] **Minor edits if needed**
  - [ ] Fix typos
  - [ ] Clarify confusing parts
  - [ ] Add examples if requested
  - [ ] Add images you didn't have time for

### First Month

- [ ] **Analyze performance**
  - [ ] Total views
  - [ ] Read ratio
  - [ ] Top traffic sources
  - [ ] Most highlighted sections

- [ ] **Repurpose content**
  - [ ] Tweet thread with key points
  - [ ] LinkedIn carousel
  - [ ] Dev.to cross-post
  - [ ] YouTube video (optional)

- [ ] **Plan next article**
  - [ ] What did readers want to know more about?
  - [ ] What questions came up in comments?
  - [ ] What related topics could you cover?

## üí° Tips for Better Performance

### Before Publishing

1. **Publish on Tuesday-Thursday** (best engagement)
2. **Publish at 7-9 AM or 12-2 PM** (your audience's timezone)
3. **Avoid weekends** (lower engagement generally)

### During Publishing

1. **Use all 5 tags** (maximum discoverability)
2. **Choose "Distribute to Medium"** (more reach)
3. **Enable social sharing** (immediate promotion)

### After Publishing

1. **Share within first hour** (Medium algorithm boost)
2. **Respond quickly to comments** (shows you're active)
3. **Share multiple times** (different audiences see it)
4. **Cross-post after 1-2 weeks** (Dev.to, Hashnode, etc.)

## üö´ Common Mistakes to Avoid

- ‚ùå **Not reading before publishing** ‚Üí Always proofread
- ‚ùå **Forgetting cover image** ‚Üí Required for good first impression
- ‚ùå **No tags** ‚Üí Article won't be discovered
- ‚ùå **Not sharing** ‚Üí Even great content needs promotion
- ‚ùå **Ignoring comments** ‚Üí Engagement matters for algorithm
- ‚ùå **Expecting instant success** ‚Üí Takes time to build audience
- ‚ùå **Comparing to viral posts** ‚Üí Most articles don't go viral (and that's OK)
- ‚ùå **Perfect formatting** ‚Üí Done is better than perfect
